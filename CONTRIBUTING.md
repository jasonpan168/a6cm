# 贡献指南

感谢你对 A6.cm 短网址项目的兴趣！本文档说明如何为本项目做贡献。

## 我们欢迎的贡献

- 🐛 **Bug 报告** - 报告你发现的问题
- 🎯 **功能建议** - 建议新功能或改进
- 📝 **文档改进** - 改进 README 或代码注释
- 🔧 **代码修复** - 提交修复代码的 PR
- 🌍 **翻译** - 帮助本项目国际化

## 提交 Issue

在提交 Issue 前，请先检查是否已有相同的 Issue。

### Bug 报告

请包含以下信息：
1. **问题描述** - 清晰描述问题
2. **重现步骤** - 如何重现问题
3. **预期行为** - 应该发生什么
4. **实际行为** - 实际发生了什么
5. **环境信息**
   - PHP 版本
   - 数据库类型和版本
   - 操作系统
   - Web 服务器
6. **错误日志** - 相关的错误消息

### 功能请求

请说明：
1. **问题背景** - 为什么需要这个功能
2. **建议的解决方案** - 你的想法
3. **其他替代方案** - 其他可能的方案

## 提交代码修改

### Fork 和克隆

```bash
# Fork 项目到你的账号
# 克隆你的 fork
git clone https://github.com/jasonpan168/a6cm.git
cd a6cm

# 添加上游 remote
git remote add upstream https://github.com/jasonpan168/a6cm.git
```

### 创建分支

```bash
# 从最新的 main 分支创建新分支
git fetch upstream
git checkout -b feature/your-feature-name upstream/main

# 或修复 bug
git checkout -b fix/bug-description upstream/main
```

### 开发指南

1. **代码风格**
   - 遵循项目现有的代码风格
   - 使用 4 空格缩进
   - 变量名使用有意义的英文或中文
   - 为复杂逻辑添加注释

2. **提交消息**
   - 第一行是简洁的标题（不超过 50 字符）
   - 空一行
   - 后面是详细说明（推荐）
   - 使用英文或中文，保持一致

   ```
   Fix: 修复用户登录时的验证问题

   - 当密码为空时仍然通过验证
   - 添加了密码非空的检查
   - 更新了错误提示信息
   ```

3. **测试你的修改**
   - 在本地充分测试
   - 检查没有破坏现有功能
   - 对于新功能，最好添加使用示例

4. **数据库修改**
   - 如果修改了数据库结构，更新 `create_tables.sql`
   - 记录 migration 步骤

### 提交 Pull Request

1. **推送你的分支**
```bash
git push origin feature/your-feature-name
```

2. **创建 Pull Request**
   - 访问项目 GitHub 页面
   - 点击"Pull Request"
   - 使用清晰的标题和描述
   - 关联相关的 Issue（如有）

3. **PR 描述模板**
```markdown
## 描述
简洁描述你做的改动

## 相关 Issue
关闭 #123

## 修改类型
- [ ] Bug 修复
- [ ] 新功能
- [ ] 文档更新
- [ ] 其他

## 修改清单
- [ ] 我已测试这些修改
- [ ] 我已更新相关文档
- [ ] 我的代码遵循项目的代码风格
- [ ] 我没有破坏任何现有功能
```

## 代码审查流程

- 维护者会审查你的 PR
- 可能会要求修改或提出问题
- 一旦批准，将被合并到主分支
